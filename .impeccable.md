# Design Context

This project is a native desktop video editor moving from Dioxus to egui. The UI should feel like a precise, professional editing tool rather than a marketing surface: compact, dark, quiet, scannable, and built for repeated production workflows.

The Dioxus implementation is the visual and behavioral anchor during the migration, but the egui version should use reusable toolkit-level patterns instead of one-off widget tweaks. Preserve the proven information architecture, layout flow, and timeline precision while allowing the component styling to evolve into a coherent egui-native design system.

## Current Design Principles

- Prefer reusable ui-kit templates, constants, and component helpers over hardcoded per-screen values.
- Keep dark surfaces layered but avoid pure black or pure white for fields, outlines, and text.
- Use consistent field, button, card, and panel dimensions from shared tokens.
- Use responsive layout primitives where possible; reserve manual rect math for precision tools such as timelines, rulers, tracks, and exact button rows.
- Modal and floating surfaces must have coherent rounded corners, padding, close buttons, and backdrop treatment.
- Form fields should share height, fill, outline, text alignment, focus highlighting, and selection behavior unless there is a deliberate component variant.
- Dense editor panels should align to predictable grids, with consistent gutters and no accidental drift.

## Timeline Direction

The timeline is a precision editing surface. It should prioritize deterministic geometry, clear scale, and direct manipulation:

- The Dioxus timeline is the main reference for header structure, track layout, ruler/ticks, clip blocks, markers, playhead styling, and media previews.
- Clip width and position must be derived from time, fps, and pixels-per-second consistently.
- Keep fixed track labels and the horizontally scrollable ruler/track canvas as separate regions, and clip all scrollable paint to the canvas viewport so media blocks never bleed into the label rail.
- At high zoom levels, frame ticks should appear and playhead/clip operations should snap to frame boundaries.
- Interaction precedence should match editor expectations: clip edge resize, clip body move, marker/clip selection, ruler or empty-space playhead seek.
- Track rows need clear horizontal dividers, compact spacing, and readable labels.
- Media clips should show thumbnails or waveform previews when cached media data is available.
- Timeline thumbnail strips should use time-indexed cached thumbnails across the clip, falling back gracefully only when a tile-specific frame is unavailable.
- Timeline controls and hotkeys should expose the same operations: zoom out/in, fit, frame scale, playback buttons, timecode, collapse/expand, horizontal scroll with Shift+wheel, and plus/minus zoom keys.
