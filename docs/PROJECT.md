# Project Status

LatentSlate is a Windows-first functional alpha for local AI video workflows, created by Enviral Design. It combines a Rust/egui desktop timeline, project-local media management, FFmpeg-backed preview/export, audio playback, generative asset versioning, and bring-your-own ComfyUI providers.

This file is the living project status page. Keep it current, concise, and factual.

## Current Positioning

- **Audience:** technically capable ComfyUI users, AI video creators, Rust contributors, and OSS reviewers.
- **Project shape:** local-first generative NLE, not a hosted generation service.
- **Primary provider path:** local ComfyUI workflows exported as API JSON.
- **Platform:** Windows is the only actively developed and tested platform today.
- **Release state:** no installer or stable binary release yet.
- **Readiness:** useful for experimentation and contribution, not production-ready.

## What Works Today

- Native desktop shell built with `egui`/`eframe`.
- LatentSlate app icon is embedded for the Windows taskbar and title/headerbar window icon.
- Project create/open/save with self-contained project folders.
- Project-local asset import for images, audio, and video.
- Timeline with video, audio, and marker tracks.
- Clip selection, moving, trimming/resizing, snapping, labels, transforms, opacity, and track-level controls.
- Preview panel with FFmpeg-backed media decode, cached timeline thumbnails, transform handles, preview stats, and scrub/playhead updates.
- Audio import, waveform rendering, timeline playback, audio scrubbing, and clip/track volume controls.
- Generative image, video, and audio assets with per-asset `config.json`, active versions, and version history.
- Generation queue with provider selection, schema-driven inputs, progress/status feedback, cancellation, and generated output saved into the project.
- Timeline and Asset Lab continuation actions seed new I2I/I2V/edit steps from the referenced generative settings while replacing the media reference; last-frame I2V extension continues on the source track.
- Error-like app status messages are emphasized in the bottom status bar instead of rendering as muted chrome text.
- ComfyUI provider execution through API workflow JSON plus embedded manifest bindings generated by the Provider Builder.
- Provider Builder UI for ComfyUI workflows: workflow picker, node browser, exposed inputs, output node selection, and optional live ComfyUI schema refresh.
- Writable app data defaults to `LatentSlateData/` beside the running executable, with `LATENTSLATE_HOME` available as an explicit override.
- Provider JSON stores cloud API keys and embedded ComfyUI manifest bindings; `LatentSlateData/workflows/` is available for local workflow JSON files.
- Experimental OpenAI image, xAI image, and xAI Grok video adapters using provider JSON API keys.
- MP4 export through FFmpeg with H.264/H.265 options, quality presets, optional audio mixdown, timestamp overlay option, and progress UI.
- Rust-native Agent API / automation harness for smoke scenarios and local AI operation.
- Top-bar Agent API popover with localhost enable/disable, port display, and copyable agent primer.
- Self-documenting Agent API routes for help, schema, capabilities, state, jobs, command execution, export status, generation waits, and rendered captures.
- Agent visual feedback through normal and enhanced timeline/clip/asset frame captures and cut sheets saved under ignored `LatentSlateData/tmp/agent-captures`, cleared on app startup.
- Agent-accessible provider metadata, provider/input descriptions, provider JSON edits, generative config edits, queue operations, project settings, timeline edits, batch clip moves, and export control.

## Current Limitations

- UI, project format, and provider manifest details can still change.
- FFmpeg setup is still manual for developers/adventurous users.
- ComfyUI custom nodes, model files, and workflow dependencies are not bundled.
- ComfyUI workflow compatibility must be validated per workflow.
- `CustomHttp`, fal.ai, Replicate, Veo, and similar adapters are not implemented.
- Export exists but needs broader real-world validation and better diagnostics.
- Test coverage is small.
- `cargo clippy --all-targets -- -D warnings` is not clean yet and is not a CI gate.
- The Agent API is localhost-only development/control infrastructure, not a network-exposed remote-control service or MCP server yet.
- macOS and Linux are future work, not supported release targets.

## Roadmap

Near-term:

- Replace placeholder screenshots with a sanitized demo project and real walkthrough captures.
- Harden export with clearer errors, broader source-media coverage, and documented FFmpeg requirements.
- Improve ComfyUI video workflow compatibility for T2V, I2V, V2V, and first/last-frame workflows.
- Improve Provider Builder drift handling when workflow node IDs change.
- Add focused tests around project persistence, provider manifests, generation input resolution, timeline operations, and export.
- Clean enough clippy lint debt to add clippy to CI.

Medium-term:

- Package a Windows release.
- Add more provider adapters based on actual user/contributor demand.
- Build a small compatibility catalog for known-good ComfyUI workflows.
- Investigate macOS and Linux build requirements.
- Improve public demo assets and onboarding docs.

Not a primary goal:

- Replacing Premiere, Resolve, or a full traditional NLE.
- Bundling ComfyUI, models, or arbitrary custom nodes.
- Running a hosted provider marketplace before the local editor is stable.

## Core Decisions

| Area | Decision |
|---|---|
| Project identity | Public name, executable, package, automation env vars, and portable app data root use `LatentSlate`; maker identity is Enviral Design. |
| UI runtime | `egui`/`eframe` native desktop shell. |
| Language | Rust. |
| Project model | Project equals folder; imported media is copied into project-local folders. |
| Timeline model | Video/audio clips are range-based; markers are point-based. |
| Image references | Images live on video tracks as still clips or keyframe-reference display clips; there is no separate keyframe track. |
| Generative assets | Explicit assets under UUID-keyed `generated/{image,video,audio}/...` folders with config, active version, and version history. Hollow assets do not fall back to leftover files. |
| Provider model | Single-purpose provider entries grouped by output type; input schema defines workflow intent. |
| ComfyUI binding | Embedded provider manifests bind exposed fields to ComfyUI node IDs plus input keys. |
| Provider API keys | Cloud provider API keys are stored inline in provider JSON and redacted from API state responses. |
| Export | FFmpeg-backed MP4 export for now; bundling FFmpeg is future work. |
| Agent API | Loopback-only Agent API drives real editor/UI paths for smoke checks, local AI operation, generation waits, and rendered visual capture. |
